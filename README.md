# ProjetHTML


1. Introduction

Ce document décrit les spécifications du projet HTML ainsi que les critères d’évaluation.

2. Spécifications

Vous devez créer les pages HTML d’un site web présentant différents pays.

Chaque membre de votre équipe rédigera une page HTML du site et présentera un pays différent sur sa page.

Par exemple, vous pouvez inclure sur votre page HTML :

- L’image du drapeau du pays
- Les caractéristiques du pays (langues, monnaie, lieux emblématiques, etc.)
- Un formulaire pour laisser un commentaire

Détails des spécifications :

1. Il n’y a pas de page d’accueil pour ce site. Il y aura simplement une page par membre de l’équipe.
2. Le site aura un en-tête et un menu communs à toutes les pages. Cela devra être défini et réalisé en équipe.
3. Le menu donnera accès à la page de chaque membre de l’équipe. Il devra avoir un effet dynamique (hover).
4. Le nom de tous les membres de l’équipe devra apparaître en bas de chaque page. Cela devra être défini et réalisé en équipe.
5. La feuille de style CSS doit être commune à toutes les pages du site. Cela devra être défini et réalisé en équipe.
6. Chaque page doit présenter un pays différent.
7. Chaque membre devra créer sa propre page HTML contenant les éléments suivants (ces éléments ne font pas partie de la section commune) :

   - Une table
   - Une image
   - Un formulaire avec au moins deux champs
   - Une liste
   - Un lien externe
8. Le nom du fichier HTML de chaque membre doit être composé de son prénom et nom, séparés par un underscore.

   Exemple : `olivier_choquet.html`
9. Chaque membre devra ajouter un identifiant (`id`) ou une classe (`class`) à sa page HTML et définir un style pour cet élément.
10. Méthodologie

C’est un travail en équipe : il est important de discuter et de prendre les décisions ensemble pour les parties communes.

Le site doit avoir une apparence cohérente.

Un aspect essentiel du projet est le partage du code et la communication entre les membres.

Définissez ensemble comment vous allez partager le code (OneDrive, etc.) et comment vous allez communiquer.

Pensez aussi à la manière dont vous allez intégrer les différentes parties du code.

4. Critères d’évaluation

Voici les critères qui seront pris en compte pour votre note :

1. Respect des spécifications
2. Respect des recommandations HTML5/CSS3

   - Bonne utilisation des balises HTML5
   - Une feuille CSS externe
3. Le code et les commentaires doivent être rédigés en anglais

   - Le contenu peut être écrit dans n’importe quelle langue
4. Validation du code via [validator.w3.org](http://validator.w3.org/) → très important !

   - Si la validation échoue → minimum -5 points sur 20
5. Lisibilité du code (indentation, clarté)
6. Orthographe
7. Respect des délais (dépôt sur MooVin)
8. Aucun code JavaScript

   - Bootstrap et Tailwind sont autorisés
9. Le pied de page doit contenir le prénom et le nom de tous les membres de l’équipe
10. Dans le fichier CSS, chaque membre doit écrire son prénom et nom en haut de sa section CSS
